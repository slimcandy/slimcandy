Smart people invented perfect text css years ago.

## Paragraph

Tim Brown [described the idea](https://tbrown.org/notes/2012/02/03/molten-leading-or-fluid-line-height/) of changing `line-height` dynamically according to page
width. His pseudo-code:

```css
p {
  max-width: 35rem;
  min-width: 21rem;
  -max-line-height: 1.4;
  -min-line-height: 1.3;
}
```

After couple of years [community helped to make it come true](https://blog.typekit.com/2016/08/17/flexible-typography-with-css-locks/). CSS lock:

```css
--max-line-height: 1.4;
--min-line-height: 1.3;
--max-text-width: 35;
--min-text-width: 21;

line-height: calc(
  (var(--min-line-height) * 1em) + (var(--max-line-height) - var(--min-line-height)) *
    ((100vw - (var(--min-text-width) * 1em)) / (var(--max-text-width) - var(--min-text-width)))
);
```

[Formula explanation](https://blog.typekit.com/2016/08/17/flexible-typography-with-css-locks/). Jason Pamental also
[explains that](https://rwt.io/typography-tips/part-2-making-typography-responsive-and-laying-foundations-more-come).

Adding first letter capitalization and highlighting first line also makes text book-like. Jason Pamental
[shows examples](https://rwt.io/typography-tips/part-3-adding-web-fonts-giving-voice-our-words). Using `font-variation-settings` variations in `:root` makes it
available to change font weight dynamically:

```css
font-variation-settings: 'wght' var(--text-wght), 'ital' var(--text-ital);

p:first-of-type:first-line {
  --text-wght: 800;

  @media screen and (min-width: 43.75em) {
    --text-wght: 900;
  }
}

p:first-of-type:first-letter {
  --text-wght: 700;
  float: left;
  font-size: 2.7em;
  @media screen and (min-width: 43.75em) {
    --text-wght: 800;
  }
  @media screen and (min-width: 60.25em) {
    --text-wght: 900;
  }
}
```

## Quotes

[Ian Yates](https://webdesign.tutsplus.com/articles/start-using-quotation-marks-the-correct-way--webdesign-16905) and
[–ê—Ä—Ç–µ–º–∏–π –õ–µ–±–µ–¥–µ–≤](https://www.artlebedev.ru/kovodstvo/sections/104/) pay our attention on importance of correct quotes. Visible keys of modern keyboard is
limited to universal signs. But books use their own sutable quotes. <q>Example quotation</q>.

```css
blockquote {
  /* Native quotes */
  quotes: '\00AB''\00BB''\201E''\201C';
}

blockquote:before {
  /* Native quotes */
  content: open-quote;
}
blockquote:after {
  /* Native quotes */
  content: close-quote;
}
```

## Miscellaneous

A few little things that will make your text as perfect as the books. w3.org shows examples of
[html content structure](https://www.w3.org/WAI/tutorials/page-structure/content/). –ê—Ä—Ç–µ–º–∏–π –õ–µ–±–µ–¥–µ–≤ also has posts about
[hanging punctuation](https://www.artlebedev.ru/kovodstvo/sections/120/) and [proper signs](https://www.artlebedev.ru/kovodstvo/sections/62/). His team also
made a [Typograph tool](https://www.artlebedev.ru/typograf/) to add non-breakable spaces, dashes and utf symbols. MDN docs about
[hyphens](https://developer.mozilla.org/en-US/docs/Web/CSS/hyphens) - don't forget it needs `lang` attribute.

```css
p {
  /* Safari only hanging punctuation */
  hanging-punctuation: first force-end last;
  font-variant-ligatures: normal;
  /* First load native font, than custom one */
  text-rendering: geometricPrecision;
  /* Book-like hyphens */
  hyphens: auto;

  text-align: justify;
}
```
